# ADR-0001: the fleet catalog is a registry-generated static page at the org root

## Status
Accepted.

## Context
The fleet had demos, operator quickstarts and business docs per
vertical, plus an org profile README — but no marketplace face: no
single page listing every implemented actor with its repo and demo.
The obvious candidates were (a) itonami.cloud (gftdcojp's operator
cockpit on Cloudflare Pages) and (b) a static page on the org's own
GitHub Pages host.

## Decision
Serve the catalog at <https://cloud-itonami.github.io/> (this repo,
the org root site), generated from kotoba-lang/industry's registry.edn
at build time. Rationale:
- The venture demos already live under the SAME host
  (/cloud-itonami-isic-*/) — the root page completes the URL topology.
- The registry is the machine-readable SSoT for maturity/repo/demo;
  generating from it keeps the catalog honest by construction, exactly
  like the demo pages are generated by running the actual actors.
- itonami.cloud is a tenant-graph operator cockpit (activity → decision
  → effect → audit), not a catalog; putting a marketplace page inside
  it is a cockpit design decision for gftdcojp's own repo. Deferred:
  when itonami.cloud grows a marketplace surface, it should CONSUME the
  same registry, not replace it.

## Consequences
- Regeneration is manual/scheduled (rerun web/generate.cljs when the
  registry advances); the count badge and rows are never hand-edited.
- Org profile README links here; the catalog links back to each repo
  and demo.
