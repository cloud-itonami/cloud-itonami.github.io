Sales wedge — 購入・運用支援する旗艦業種

cloud-itonami が直接販売・運用支援するのは下記の旗艦業種（Indeed 型 / kaonavi 型 / 地域雇用型）です。狭い楔(wedge)としてここに集中し、残りの実装は下記 Library（fork 自由・直接販売対象外）として公開します。

Meta Job Search — Indeed 型求人アグリゲーターの置き換え

掲載拒否理由(募集終了・賃金不一致・転載許諾・差別広告)を実判定つきで公開。訂正・取下げは番号つき記録。職業安定法5条の4 をソフトウェアとして実装。6 法域対応。

Talent Board — kaonavi 型 HR SaaS の置き換え

保護属性は評価・配置転換の根拠にできず、帳票は目的に許された列のみ。人材データは自分の Store に残る。配置転換は常時人間承認 + from/to/承認者の台帳化。

Placement Desk — 地域雇用・人材紹介エージェンシーの置き換え

候補者 intake・マッチング・紹介手数料の完全性(手数料 = 記録された年俸 × 料率)を実判定。配置・手数料確定は常時人間承認 + from/to/承認者の台帳化。法域別の差別禁止・就労資格アセスメント。

operator になるには

  1. vertical を選び、デモとテストスイートを動かす
  2. その repo の docs/operator-quickstart.md に従って fork → 自分のデータ → 自分のデプロイ
  3. 価格の形・認証ラダー(itonami.cloud) は docs/business-model.md に — 旗艦3業種は手書きの domain-deep 版、他の actor は registry+blueprint からの生成 baseline（ユニットエコノミクスは illustrative・計測値ではない）
  4. 認証・managed 提供・導入支援に関心があれば、各 repo の operator-interest issue から連絡(例: Meta Job Search / Talent Board / Placement Desk)
